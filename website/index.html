Proximity Intelligence

Experimental proximity analytics for team cohesion, crossfire, and engagement density. This view will activate once the proximity pipeline is fully connected.

Prototype mode: waiting for proximity data ingestion.
How to read these stats Toggle guide

Metrics come from sub-second Lua proximity telemetry, then parser aggregation into engagements, teamplay, and trade/support summaries. Scope filters (session, map, round) always control what is counted.

Summary Cards
Engagements = detected combat windows.
Avg Distance = target movement during those windows.
Crossfire Events = engagements with overlapping pressure.
Hot Zones = map bins with concentrated kill activity.
Team Dynamics
Synergy Duos combine crossfire volume and delay.
Best Sync tracks lower teammate delay (ms).
Focus Survivors = survival rate when under focus.
Support Uptime = % time within support radius.
Formula + Capture Source
Engagements = count(`combat_engagement`) from Lua `ENGAGEMENTS` output.
Spawn Reaction = avg(`player_track.time_to_first_move_ms`) from `PLAYER_TRACKS` path.
Trade Attempt Rate = `trade_attempts / trade_opportunities` from `proximity_trade_event`.
Support Uptime = `support_samples / total_samples` from `proximity_support_summary`.
Return Fire / Dodge / Support Reaction = Lua `REACTION_METRICS` first-hit deltas.
Confidence tags use sample size: High 20, Medium 8-19, Low 1-7.

Onboarding: Read This Like a Story
Built for someone new: follow the lines left right.
The 5-Step Story
1. The game makes raw files.
Lua scripts on the server write stats and timing files.
2. We pick up the files.
Ingest services watch the folders and collect new data.
3. We translate files into clean stats.
Parsers and validators turn raw text into tables.
4. Everything is stored safely.
Postgres keeps the official numbers and history.
5. The bot + website show the results.
Discord embeds and the website read the database.
