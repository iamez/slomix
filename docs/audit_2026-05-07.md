## L1+L2 — Data Ingestion + Storage

### P0
- **`bot/services/round_correlation_service.py:569+`** — Agent claimed SQLite `?` placeholders in PostgreSQL queries. **[FP]** Verified: `database_adapter.py` translates `?` → `$N` for PostgreSQL. The `?` style is intentional repo-wide convention, working in production. Dismissed.
- **`bot/automation/file_tracker.py:116-117`** — `should_process_file()` auto-marks files older than lookback window as processed without verifying actual import. If SSH sync silently fails on old file, it's never re-processed. **[VERIFY-LATER]** Real concern but high-risk to change without staging test.
- **`tools/schema_postgresql.sql`** — Migration 035 (guid_canonical) not backported to schema for fresh installs. **[VERIFIED]** `_guid_canonical` columns exist in schema but no migration tracking for fresh DB bootstrap. Requires careful staging.
- **`bot/services/round_correlation_service.py:596`** — Dynamic `f"WHERE correlation_id = ?"` claim. **[FP]** Just column name interpolation, no user input.

### P1
- **`bot/services/round_correlation_service.py:279-282`** — Strategy 2 R2 merge window (30s ≤ diff ≤ 900s) edge-case mismatch on rapid rematches. Real bug, but already addressed by canonical_id (#171/#173 in May). Mostly defused.
- **`tools/schema_postgresql.sql`** — Missing FK indexes on `round_correlations(r1_round_id, r2_round_id)`. **[VERIFIED]** Easy add-on migration.
- **`migrations/035_add_guid_canonical_columns.sql`** — Backfill race (no LOCK during UPDATE). Migration already ran in production months ago. Defused for current DB; flag for any guid_canonical re-backfill.
- **`bot/services/round_linkage_anomaly_service.py:318-320`** — Anomaly metric conflates "orphan" with "deferred-pending-linkage" rows. Diagnostics noise.

### P2
- `bot/community_stats_parser.py:152-162` — `round_match_window_minutes=45` not validated against `SESSION_GAP_MINUTES`.
- `vps_scripts/stats_discord_webhook.lua:447-448` — Buffer overflow drops payloads silently.
- `bot/services/round_correlation_service.py:189-190` — Round number outside [0,2] silently dropped, no log.
- `migrations/` — No bot startup check for migration version drift.

### P3
- `bot/services/file_tracker.py:150-151` — `LIMIT 1` missing on existence checks.
